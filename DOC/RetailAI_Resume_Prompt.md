# 面試 ERP 軟體工程師專用：RetailAI 專案履歷 Prompt 與關聯架構

這份文件為您整理了 RetailAI 智慧零售系統的完整系統架構、資料庫關聯 (ERD)，以及您可以直接複製貼上給 ChatGPT / Claude 等 AI 工具的「專屬履歷生成 Prompt」。

---

## 1. 資料庫關聯架構 (Database ERD Concept)

我們在 RetailAI 中採用了標準的關聯式資料庫 (3NF) 設計，完美契合傳統 ERP 系統強調的資料一致性與 Header-Detail (表頭與明細) 結構：

- **商品與供應鏈維度**：
  - `categories` (商品分類表)：存放類別主檔 (1對多關聯至 inventory)。
  - `suppliers` (供應商表)：存放供應商主檔 (1對多關聯至 inventory)。
  - `inventory` (庫存主檔)：系統核心，紀錄商品條碼、庫存量、警示水位 (`safety_stock`)、一週叫貨基準 (`reorder_rules`)、有效期限與動態定價。

- **交易與銷售維度 (Header-Detail 架構)**：
  - `transactions` (交易主單/表頭)：紀錄每一筆 POS 結帳的總金額、付款方式 (現金/LinePay/信用卡)、經手收銀員、時間戳記。
  - `transaction_items` (交易明細表)：紀錄該筆主單下的所有商品明細 (關聯至 `inventory`)、銷售數量、單價，並由資料庫自動計算小計。

- **營運監控維度**：
  - `alerts` (異常警示表)：紀錄低庫存、商品即將過期或客流擁擠的事件。
  - `daily_kpi` / `edge_logs`：存放邊緣運算 (YOLOv8) 客流數據與每日營收快照。

## 2. 轉化為履歷表的專用 Prompt

請直接**複製以下虛線框內的內容**，發送給任何 AI 助手（如 ChatGPT、Claude），它將為您量身打造符合 ERP 公司口味的高級履歷與 STAR 成就描述：

```text
我正在應徵「ERP 軟體工程師 (ERP Software Engineer)」職位。
請根據以下我實際獨立開發的專案資訊，為我撰寫一份符合專業履歷格式的專案描述（包含「專案簡介」、「核心功能」、「技術棧」以及以數據導向的「成就點」）。

### 1. 專案名稱
RetailAI 智慧零售管理系統 (Retail Management Ecosystem)

### 2. 系統介紹
這是一個為零售業設計的現代化 ERP 與 POS 數位轉型解決方案，整合了前端收銀終端、智慧化庫存監控、與即時營收數據儀表板。系統旨在解決傳統店鋪人工盤點效率低下、叫貨缺乏標準、以及帳目與實體庫存不一致的痛點。

### 3. 資料庫設計與關聯架構 (高度契合 ERP 領域)
- 嚴謹的關聯式資料庫 (PostgreSQL) 設計，實作經典的 Header-Detail (訂單頭與明細) 架構 (`transactions` 與 `transaction_items`)，確保交易資料的一致性與金流準確性。
- 完整正規化設計：透過 Foreign Keys 將庫存 (`inventory`) 關聯至供應商 (`suppliers`) 與商品分類 (`categories`)，實現供應鏈的資料連動。
- 使用 JSONB 彈性欄位設計進階的「一週動態叫貨基準 (`reorder_rules`)」，結合即時庫存預警。

### 4. 核心功能
- **智慧 POS 收銀系統**：串接前端響應式介面與後端資料庫，實現即時交易處理、多重付款方式結帳、與購物車小計，並同步扣除雲端庫存。
- **自動化庫存進銷存管理**：支援即時進貨管理與批次到期日紀錄。內建自動化庫存預警機制，當單一商品低於安全庫存 (`safety_stock`) 或是效期少於一天時，儀表板會立即發出補貨或報廢通知。
- **即時數據決策儀表板**：不依賴靜態假資料，直接撈取真實交易明細進行大數據彙整，計算出當日營業總額與客單價，並列出低庫存商品清單，輔助店長決策。
- **AI 模組整合架構**：系統架構已預留電腦視覺 (YOLO) 邊緣運算節點以監測排隊客流，並結合機器學習進行未來七天的銷量預測。

### 5. 開發技術 (Tech Stack)
- 前端：React / TypeScript / Vite / Tailwind CSS (提供兼具美感與極致操作流暢度的管理介面)
- 後端與資料庫：Supabase / PostgreSQL (實作複雜關聯查詢、Trigger、資料庫層級邏輯與權限管控)
- 架構與工具：Model Context Protocol (MCP) Agent 架構、RESTful API。

### 撰寫要求
1. **強化 ERP 關聯**：請針對 ERP 公司特別看重的「複雜業務邏輯理解能力」、「表頭表身架構設計」與「資料庫關聯規劃能力」進行強調。
2. **採用 STAR 法則**：請幫我發想並撰寫 3 到 4 點高含金量的成就點 (Achievements)。必須以數據或具體效益為導向。
例如：
- 「設計關聯式資料庫與 Header-Detail 架構，提升百萬筆交易明細的查詢效能，確保庫存扣除的 100% 準確性。」
- 「實作庫存水位監控邏輯，將傳統人工盤點的時間耗損降低 X%，庫存異常發現率提升 Y%...」等等。
3. **語氣專業精煉**：使用強而有力的動詞開頭，符合資深技術工程師的履歷標準。
```
